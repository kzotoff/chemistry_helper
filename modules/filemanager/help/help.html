Файловый менеджер

Общее описание

Модуль предназначен для управления пользовательскими файлами - страницами, таблицами стилей, скриптами и прочими полезными штуками.

Чтобы поуправлять файлами, нужно выбрать категорию и нажать кнопку "Перейти"

В каждую категорию можно добавить файл (только с подходящим расширением) и удалить ( ) уже существующий. В категориях, содержащих текстовые файлы (стили, скрипты, страницы) их можно редактировать ( ). Можно также переименовать файл или переместить в другую категорию, для этого нужно в поле "имя файла" ввести новый полный путь к файлу. Файл будет перемещен только при условии, что новое расширение подходит для нужной категории (то есть, скрипт с расширением .js нельзя положить в папку с таблицами стилей, только .css).

Список каталогов

В следующих каталогах можно управлять файлами (как они представлены в административной части модуля в стандартной поставке):

Каталог Описание
CSSтаблицы стилей, которые можно применять к страницам
scriptsскрипты на языке javascript
imagesизображения
pagesсобственно основной контент сайта
XSL stylesheetsXSL-трансформации, которые можно применять к страницам.
Main filesосновные файлы сайта — шаблон и таблица стилей
filesразличные файлы, предназначенные для загрузки пользователями
trashкорзина. Сюда попадают удаленные страницы

Кроме того, модули могут добавлять в этот список свои каталоги.

Популярные задачи

Редактирование основного шаблона сайта

  1. Открываем файловый менеджер.
  2. Выбираем категорию "Main files", нажимаем кнопку .
  3. В появившемся списке наводим мышку на template.html и кликаем на кнопке редактирования ( ).
  4. В открывшемся диалоговом окне видим исходный код основного шаблона сайта и можем вносить изменения.
  5. После того, как изменения внесены, жмем кнопку

Технические вопросы

javascript: в стандартном комплекте уже есть jQuery, jQueryUI, jQuery.tablesorter.

CSS: уже подключен bootstrap (только CSS, скриптовая часть отсутствует).

картинки: допустимые форматы: jpeg, png, gif.

страницы: есть некоторое количество зарезервированных названий:

  • service_feedback_ok и service_feedback_failed: модуль обратной связи будет переходить на эти страницы при успешной и неуспешной отправке сообщения. Соответственно, эти страницы необходимо создать.

XSL: ошибки трансформации не отслеживаются, выверяйте XML тщательно!

файлы: из этого каталога файлы выдаются пользователю без специальной обработки, "как есть", MIME-тип будет определяться сервером.
