-------------------------------------------------------------------------------
               BACKUP : Простой модуль для резервного копирования
-------------------------------------------------------------------------------

Краткое описание:
	модуль предназначен для работы с резервными копиями пользовательских файлов.
	возможности:
		* создание резервной копии файлов по заранее составленному списку
		* восстановление файлов из копии
		* загрузка файла копии на локальный компьютер


Использование:
	Заходим в административную часть, там кнопка "backup". Видим список
	имеющихся копий, если таковые есть, и форму для создания новой копии.

	* Для создания новой копии:
		* вводим название файла (допустимые знаки - буквы, цифры, знак
		  подчеркивания), расширение .zip или пустое (будет автоматически
		  добавлено .zip). По умолчанию используется текущая дата и время
		  в формате ГГГГММДДЧЧММСС
		* вводим описание копии (необязательный параметр)
		* Нажимаем кнопку "Создать"

	* Для восстановления данных из резервной копии:
		* Наводим курсор мыши на строку с нужной копией - появятся кнопки
		* Нажимаем кнопку "восстановить"
		* (!!!) восстанавлюваются все файлы, содержащиеся в резервной копии,
		  независимо от настроек, перезаписываются без предупреждения
		* Если какой-то файл восстановить не удалось (например, уже есть такой
		  файл с атрибутом "только для чтения"), восстановление прерывается
		* После восстановления выводится сообщение об успехе или неуспехе. При
		  неуспешном восстановлении выводится название файла, на котором
		  произошла неудача

	* Для загрузки копии с сайта на локальный компьютер:
		* Наводим курсор мыши на строку с нужной копией - появятся кнопки
		* Нажимаем кнопку "скачать"

	Загрузка на сервер собственных копий не предусмотрена в целях безопасности

Настройка:
	* по состоянию на версию 1.0 - только через настроечный файл
	* в секцию конфигурации "backup_list" вписываем список файлов, которые
	  нужно сохранять.


API:
	не предусмотрено


Дополнительные public-функции в модуле:
	нет


Управление с помошью шаблонов:
	не предусмотрено


Константы и дополнительные файлы:
	./data/  : каталог для хранения резервных копий
	form.xsl : файл XSL-трансформации для административной формы
	
	
Пример настроек:

	<config>

		<!--
			список файлов для сохранения
		-->
		<backup_list>

			<!--
				будет сохранен единичный файл в основном каталоге системы
			-->
			<elem>template.html</elem>

			<!--
				будет сохранен файл в подкаталоге
			-->
			<elem>data/cms.sqlite</elem>
			
			<!--
				будут сохранены все файлы в подкаталоге userfiles/css
			-->
			<elem>userfiles/css/*.*</elem>

		</backup_list>

	</config>
