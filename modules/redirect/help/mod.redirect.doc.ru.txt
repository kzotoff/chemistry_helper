-------------------------------------------------------------------------------
                REDIRECT : модуль для перенаправления запросов
-------------------------------------------------------------------------------

Краткое описание:
	Модуль служит для безусловного перенаправления GET-запросов


Использование:
	Вписываем в конфигурационный файл правила перенаправления.

	Правила обрабатываются последовательно в порядке описания, в случае
	совпадения сервер посылает заголовки перенаправления и скрипт завершает
	работу. Остальные модули не вызываются.

	Правила неполного совпадения требуют совпадения всех пар образцов.
	
	HTTP-код ответа определяется следующим образом в следующем порядке:
		* по умолчанию код равен 200, для избежания циклического
		  перенаправления в случае ошибки
		* если присутствует элемент "location", код заменяется на 301
		* если код задан прямо ("code" : "502") и он корректный, то посылается
		  указанный код


Настройка:
	по состоянию на версию 1.0 - только через настроечный файл


API:
	Нет


Дополнительные public-функции в модуле:
	Нет


Управление с помошью шаблонов:
	Нет

	
Константы и дополнительные файлы:
	DEFAULT_HTTP_CODE : HTTP-код, отсылаемый по умолчанию


Пример настроек:

	<config>

		<!--
			правила перенаправления
		-->
		<redirect_rules>

			<!--
				для случая, когда требуется точное совпадение строки запроса
				перенаправление будет только если вся строка запроса после "?"
				совпадает с образцом: site.com/main?key1=value1&key2=value2
			-->
			<rule>
				<!-- строка, которую ловим -->
				<check>key1=value1&key2=value2</check>

				<!-- HTTP-код ответа -->
				<code>502</code>
				
			</rule>

			<!--
				случай неполного совпадения или другой порядок параметров,
				например site.com/main?key2=value2&foo=bar&key1=value1
				тестовые пары в запросе должны присутствовать все
			-->
			<rule>

				<!-- массив пар "параметр" : "значение" -->
				<check>
					<key1>value1</key1>
					<key2>value2</key2>
				</check>

				<!-- куда перенаправляем. код автоматически будет 301 -->
				<location>now-redirect-here</location>
				
			</rule>
				
		</redirect_rules>

	</config>
